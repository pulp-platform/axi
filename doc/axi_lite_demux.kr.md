# AXI4-Lite 디멀티플렉서

`axi_lite_demux`는 하나의 AXI4-Lite 연결을 여러 개로 분리합니다.

## 설계 개요

디멀티플렉서는 하나의 *슬레이브 포트*와 설정 가능한 수의 *마스터 포트*를 가집니다.

AW 및 AR 채널은 각각 *select* 입력을 가지며, 이를 통해 요청을 전송할 마스터 포트를 결정합니다. 예를 들어, select는 주소 범위를 서로 다른 AXI4-Lite 슬레이브에 매핑하기 위한 (외부) 주소 디코더에 의해 구동될 수 있습니다.

W 채널의 비트들은 대응하는 AW 비트와 동일한 선택에 따라 디멀티플렉서에 의해 라우팅됩니다.

B 및 R 채널의 비트들은 각각의 FIFO에 저장된 스위칭 결정에 따라 마스터 포트에서 멀티플렉싱됩니다.


## 구성

이 디멀티플렉서는 다음 표에 나열된 파라미터를 통해 구성됩니다:

| Name                 | Type               | Definition                                                                                                                                                                                                                                                                                             |
|:---------------------|:-------------------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `NoMstPorts`         | `int unsigned`     | 디멀티플렉서의 AXI4-Lite 마스터 포트 수 (즉, 연결 가능한 AXI4-Lite 슬레이브 모듈의 수).                                                                                                                                                                                                              |
| `MaxTrans`           | `int unsigned`     | 슬레이브 포트에서 동시에 처리 가능한 최대 [진행 중](../doc#in-flight) 트랜잭션 수.                                                                                                                                                                                                                    |
| `FallThrough`        | `bit`              | AW 채널의 라우팅 결정이 W 채널로 폴-스루됩니다 (즉, 사이클을 소비하지 않음). 이를 활성화하면 디멀티플렉서가 AW 비트와 동일한 사이클에 W 비트를 수락할 수 있지만, `slv_aw_select_i`의 로직으로 인해 W 채널의 조합 경로가 길어집니다. |
| `SpillXX`            | `bit`              | 디멀티플렉서 이전 각 채널(AW, W, B, AR, R)에 스필 레지스터를 하나씩 삽입합니다.                                                                                                                                                                                                                        |

나머지 파라미터는 디멀티플렉서의 포트를 정의하는 타입입니다. `_*chan_t` 타입은 `axi/typedef.svh`에 정의된 `AXI_TYPEDEF` 매크로를 사용하여 구성에 맞게 바인딩되어야 합니다.

### 파이프라이닝 및 지연

`SpillXX` 파라미터를 사용하면 디멀티플렉서의 각 채널 앞에 스필 레지스터를 삽입할 수 있습니다. 스필 레지스터는 채널의 모든 조합 경로(페이로드 및 핸드셰이크 모두)를 차단합니다. 따라서 채널당 1사이클의 지연이 추가되지만 처리량에는 영향을 미치지 않습니다.

모든 `SpillXX`와 `FallThrough`가 비활성화되면, 이 멀티플렉서를 통과하는 모든 경로는 조합 회로로 구성됩니다 (즉, 지연이 없습니다).

## 포트

| Name                              | Description                                                                                                                                                                                      |
|:----------------------------------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `clk_i`                           | 다른 모든 신호(`rst_ni` 제외)가 동기화되는 클럭.                                                                                                                                                |
| `rst_ni`                          | 리셋, 비동기, 액티브-로우.                                                                                                                                                                       |
| `test_i`                          | 테스트 모드 활성화 (액티브-하이).                                                                                                                                                                |
| `slv_*` (except `slv_*_select_i`) | 디멀티플렉서의 단일 슬레이브 포트.                                                                                                                                                               |
| `slv_{aw,ar}_select_i`            | 쓰기 또는 읽기 요청이 각각 디멀티플렉싱될 마스터 포트의 인덱스. 이 신호는 AW 또는 AR 채널에서 핸드셰이크가 [보류 중](../doc#pending)인 동안 안정적으로 유지되어야 합니다.                          |
| `mst_*`                           | 디멀티플렉서의 마스터 포트 배열. 각 포트의 배열 인덱스가 마스터 포트의 인덱스입니다.                                                                                                             |


### 구현

W 비트들은 해당 AW에 대한 `slv_aw_select_i` 값에 의해 정의된 마스터 포트로 라우팅됩니다. W 트랜잭션의 순서는 AW의 순서에 의해 결정되므로, select 신호들은 FIFO에 저장됩니다. 이 FIFO는 AW 슬레이브 채널에서 핸드셰이크가 발생할 때 푸시되고, 해당 W 트랜잭션의 핸드셰이크가 완료될 때 팝됩니다.

디멀티플렉서는 응답 라우팅을 위해 각각의 FIFO에 라우팅 결정을 저장합니다.
